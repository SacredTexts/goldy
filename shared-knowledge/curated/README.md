# Curated Knowledge — Contribution Guide

This directory contains developer-curated lessons learned from coding-research interviews. These are manually exported insights — not raw data dumps.

## How to Contribute

1. After a particularly insightful coding-research interview, create a markdown file here
2. Use the template below
3. Submit via PR to the goldy repo

## Template

```markdown
---
date: YYYY-MM-DD
author: your-name
project: project-name
track: bug_issue | feature_build | ui_surface | performance | schema_db | refactor_architecture
---

# Lesson: [Brief title]

## What happened
[1-2 sentences describing the situation]

## What the interview surfaced
[What questions revealed the real problem vs the stated problem]

## Key takeaway
[The insight that should inform future interviews]
```

## Rules

- No raw session dumps — curate before submitting
- No sensitive data (API keys, credentials, internal URLs)
- Focus on reusable insights, not project-specific details
- Keep entries concise (under 200 words)
- Must be reviewed before merge
